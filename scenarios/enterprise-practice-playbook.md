# 企业级 LLM 实践方案集

这份方案集把六个大节点转成企业可执行的落地场景。每个场景都要求有数据支撑：先建立基线，再定义目标指标、采集口径、验收方式和风险控制。表中的数字不预设结果，只定义应该采集和比较的数据。

细节、资料地址、操作步骤与最小字段以细分文档为准；本页只负责选题、风险分级和指标口径。

## 使用方式

- 先选一个业务流程，不要从模型能力出发。
- 先记录当前人工流程的基线数据，再引入 LLM。
- 每个试点必须有业务负责人、技术负责人、数据口径负责人和风险审批人。
- 试点周期建议 2-4 周，至少覆盖 50 条真实或脱敏样例。
- 只有当质量、效率、成本和风险指标都可解释时，才进入扩大试点。

## 试点选型决策

按风险与数据就绪度选题，避免一上来做可写生产变更的 Agent。

| 风险等级 | 典型场景 | 数据就绪条件 | 建议 |
| --- | --- | --- | --- |
| L1 只读 / 草稿 | Prompt 运营、制度问答、合同摘要草稿 | 有脱敏样例、人工质检口径、可引用资料版本 | 优先试点 |
| L2 检索增强 | Context / RAG、销售知识助手 | 有权限模型、版本字段、引用评测集 | 在 L1 稳定后启动 |
| L3 工具接入 | MCP 只读查询、指标 Skill、文档生成 | 有工具 schema、审计日志、审批人 | 先只读，再开放建议型写入 |
| L4 多步 Agent | 缺陷修复、运维诊断、客户成功行动计划 | 有 trace、停止条件、人工接管、回滚路径 | 必须先过治理门禁 |
| L5 自动改进 | Improvement Loop、自动修知识库 / Prompt | 有回归集、版本回滚、失败分类 | 仅在 L2–L4 可解释后启用 |

治理与安全贯穿所有等级，立项前先读 [企业治理与安全场景](enterprise-governance-safety.md)。

## 数据记录模板

| 字段 | 说明 |
| --- | --- |
| 业务流程 | 说明该场景嵌入哪个真实流程，例如客服、法务、研发、销售、运维 |
| 风险等级 | L1–L5，决定审批深度和是否允许自动执行 |
| 基线数据 | 试点前人工流程的耗时、质量、成本、风险事件或满意度 |
| 样例来源 | 工单、合同、日志、代码、会议纪要、CRM、报销单等 |
| 样例规模 | 试点样例数量、抽样方式、脱敏方式和覆盖业务类型 |
| LLM 介入点 | 生成、抽取、分类、检索、工具调用、评测或修复 |
| 质量指标 | 正确率、召回率、引用准确率、质检通过率、人工采纳率 |
| 效率指标 | 平均处理时长、等待时间、返工次数、自动完成比例 |
| 成本指标 | 模型调用成本、人工复核成本、工具运行成本 |
| 风险指标 | 幻觉率、越权访问次数、敏感信息泄露、错误升级或错误拒答 |
| 决策结论 | 扩大试点、继续观察、回滚、补数据或改流程 |
