# 企业级 LLM 实践方案集

这份方案集把六个大节点转成企业可执行的落地场景。每个场景都要求有数据支撑：先建立基线，再定义目标指标、采集口径、验收方式和风险控制。表中的数字不预设结果，只定义应该采集和比较的数据。

## 使用方式

- 先选一个业务流程，不要从模型能力出发。
- 先记录当前人工流程的基线数据，再引入 LLM。
- 每个试点必须有业务负责人、技术负责人、数据口径负责人和风险审批人。
- 试点周期建议 2-4 周，至少覆盖 50 条真实或脱敏样例。
- 只有当质量、效率、成本和风险指标都可解释时，才进入扩大试点。

## Prompt 工程实践

| 场景 | 企业操作方案 | 数据支撑与验收指标 | 风险控制 |
| --- | --- | --- | --- |
| 客服答复标准化 | 选取高频问题，建立回复 prompt 模板，要求模型输出答案、引用依据、升级条件和禁用话术 | 基线：人工平均处理时长、一次解决率、质检通过率；验收：处理时长下降、质检通过率不低于人工基线、升级判断召回率可解释 | 禁止模型承诺政策外权益；高投诉风险问题必须转人工 |
| 销售邮件生成 | 按客户行业、阶段、历史互动生成邮件草稿，销售只做确认和个性化补充 | 基线：邮件撰写时长、回复率、预约率；验收：节省撰写时间且回复率不下降；抽样检查事实错误率 | 禁止虚构客户案例、价格和承诺；敏感客户必须人工确认 |
| 合同条款摘要 | 对标准合同提取付款、交付、违约、保密、终止条款，输出固定字段摘要 | 基线：法务初审时长、字段遗漏率；验收：关键字段召回率、摘要准确率、人工复核节省时间 | 不替代法律意见；非标准条款必须标红并要求人工复核 |
| HR 简历初筛说明 | 对简历与岗位要求做匹配说明，只输出证据和待确认问题，不做自动淘汰 | 基线：筛选耗时、复核一致率；验收：证据命中率、面试官认可率、候选人公平性抽检 | 禁止使用性别、年龄、地域等敏感特征；保留人工决策 |
| 财务报销说明生成 | 根据报销制度和票据字段生成缺失材料、违规原因和补交建议 | 基线：退回率、人工审核时长；验收：规则命中准确率、员工二次提交通过率 | 金额异常、重复票据和高风险供应商必须走规则系统或人工审核 |
| 内部知识问答口径 | 把制度问答 prompt 固化为“答案、依据、适用范围、不确定点、转人工条件” | 基线：员工咨询量、HR/IT 响应时长；验收：答案引用率、错误升级率、重复咨询下降 | 所有答案必须引用制度版本；过期制度不得进入上下文 |

## Context 工程实践

| 场景 | 企业操作方案 | 数据支撑与验收指标 | 风险控制 |
| --- | --- | --- | --- |
| 制度知识库问答 | 将制度文档切片，按部门、地区、版本和生效日期建立检索索引，回答时返回引用 | 基线：人工咨询响应时长、答案错误率；验收：引用命中率、过期制度拦截率、员工满意度 | 检索结果必须带版本；冲突制度必须提示人工确认 |
| 售后工单上下文装配 | 将客户资料、历史工单、产品手册、SLA 和当前问题装配到同一回答上下文 | 基线：首次响应时长、重复追问次数；验收：一次解决率、上下文命中率、误引用率 | 客户隐私字段脱敏；跨客户资料不得混入上下文 |
| 研发代码库问答 | 以代码文件、README、测试、Issue 和提交记录作为上下文来源，回答架构和变更问题 | 基线：新人定位文件耗时、重复问答数量；验收：文件引用准确率、建议可执行率 | 不把未审查代码建议直接合并；生成内容需开发复核 |
| 销售知识助手 | 根据产品资料、竞品对比、价格政策和客户行业资料组合上下文 | 基线：销售查资料时长、方案准备时长；验收：资料引用覆盖率、错误事实率、方案生成时间 | 价格政策和承诺范围必须引用官方材料 |
| 会议决策追踪 | 把会议纪要、任务系统、相关文档和历史决策压缩成下一次会议上下文 | 基线：会前准备时长、遗漏行动项数量；验收：行动项召回率、决策追溯率 | 决策变更必须保留原始来源，不用摘要覆盖原文 |
| 投研资料整合 | 对公告、研报、财报、新闻和内部笔记做来源分层，生成可追溯摘要 | 基线：研究员整理资料时长；验收：引用覆盖率、事实核验通过率、来源等级分布 | 市场观点与事实分离；禁止把二手报道当一手事实 |

## MCP + Skill 实践

| 场景 | 企业操作方案 | 数据支撑与验收指标 | 风险控制 |
| --- | --- | --- | --- |
| 工单系统 MCP 接入 | 暴露只读查询、状态更新、评论追加等工具，敏感动作要求审批 | 基线：工单查询和更新耗时；验收：工具调用成功率、参数错误率、审批覆盖率 | 状态变更必须记录操作者、模型建议和人工确认 |
| 数据指标查询 Skill | 封装常用指标口径、SQL 模板和图表说明，让分析助手按固定口径查询 | 基线：临时取数等待时间、口径争议次数；验收：查询复用率、口径一致率、SQL 审核通过率 | 生产库只读；敏感字段脱敏；高成本查询限流 |
| PR Review Skill | 把代码审查规则、风险分类、测试要求和输出格式封装为 Skill | 基线：人工 review 时长、漏检缺陷数；验收：有效评论率、误报率、缺陷召回率 | 不自动通过 PR；安全和数据变更必须人工复核 |
| 发布检查 Skill | 接入 CI、变更单、监控和回滚文档，生成发布前检查清单 | 基线：发布准备时长、遗漏检查项数量；验收：检查项覆盖率、失败阻断率、回滚资料完整性 | 高风险发布必须人工审批；禁止模型绕过门禁 |
| 客户资料读取工具 | 通过 MCP 只读访问 CRM 客户档案、沟通记录和合同摘要 | 基线：销售查客户资料时长；验收：字段命中率、无权限访问拦截率 | 权限按销售归属和客户等级控制；访问日志可审计 |
| 文档生成 Skill | 封装周报、复盘、需求评审、项目状态报告模板和检查清单 | 基线：文档准备时长、格式返工次数；验收：模板字段完整率、人工修改比例 | 自动生成内容必须标记来源和未确认假设 |

## Agent 工程实践

| 场景 | 企业操作方案 | 数据支撑与验收指标 | 风险控制 |
| --- | --- | --- | --- |
| 缺陷修复 Agent | 读取 bug 描述、日志、代码和测试，生成修复计划、补丁和验证记录 | 基线：平均修复时长、回归失败率；验收：测试通过率、人工修改比例、缺陷复开率 | 只能在分支提交；禁止直接合并；安全模块需人工 review |
| 运维诊断 Agent | 读取告警、日志、指标和变更记录，输出假设、验证步骤和处置建议 | 基线：MTTA、MTTR、误报率；验收：定位准确率、建议采纳率、恢复时间变化 | 生产操作只给建议；执行重启、扩容、回滚需人工确认 |
| 数据分析 Agent | 接收业务问题，生成分析计划、查询数据、输出图表和结论 | 基线：分析交付周期、返工率；验收：查询可复现率、结论被采纳率、口径错误率 | 查询权限受控；结论必须附数据来源和假设 |
| 招投标资料 Agent | 汇总招标文件、公司资质、案例和报价约束，生成响应清单和初稿 | 基线：标书准备周期、漏项数量；验收：条款覆盖率、人工修订比例、合规问题数 | 报价和承诺必须人工确认；历史案例需授权使用 |
| 客户成功 Agent | 根据客户使用数据、工单和续约日期生成健康度分析和行动计划 | 基线：客户经理准备时长、续约风险发现率；验收：风险识别准确率、行动项完成率 | 不自动发送客户沟通；敏感客户需经理确认 |
| 知识库维护 Agent | 根据新增文档、工单和反馈识别知识缺口，提出更新 PR | 基线：知识库过期条目数量、重复咨询量；验收：更新建议采纳率、过期内容下降 | 不直接覆盖正式文档；引用和变更原因必须完整 |

## Harness 工程实践

| 场景 | 企业操作方案 | 数据支撑与验收指标 | 风险控制 |
| --- | --- | --- | --- |
| 客服机器人回归集 | 从真实工单抽样建立分类、答复、升级和拒答样例集 | 基线：人工质检分；验收：正确率、升级召回率、拒答准确率、幻觉率 | 每次上线前跑固定集；高风险类别单独设门槛 |
| RAG 引用评测 | 建立问题、标准答案、必须引用文档和禁止引用文档的评测集 | 基线：当前系统引用命中率；验收：引用准确率、证据覆盖率、无依据回答率 | 缺少证据时必须回答不确定 |
| Agent 工具调用评测 | 固定多步任务，记录工具选择、参数、失败恢复和最终产物 | 基线：人工流程耗时和成功率；验收：工具调用成功率、错误恢复率、人工接管率 | 高风险工具单独评测，不用平均分掩盖 |
| 模型替换评估 | 在相同样例、相同上下文和相同 grader 下比较新旧模型 | 基线：旧模型质量、成本、延迟；验收：质量不下降、成本延迟可接受、风险样例通过 | 不只看总分；必须看高价值客户和高风险业务样例 |
| LLM-as-judge 校准 | 用人工标注样例校准模型裁判，比较一致率和偏差 | 基线：人工一致率；验收：模型裁判与人工一致率、偏差类型、置信区间 | 关键决策不只依赖模型裁判 |
| 线上反馈闭环评测 | 将用户差评、人工纠错和投诉样例转入回归集 | 基线：重复问题出现次数；验收：同类错误复现率下降、修复验证通过率 | 反馈样例需脱敏；异常个案不直接代表整体趋势 |

## Loop 工程实践

| 场景 | 企业操作方案 | 数据支撑与验收指标 | 风险控制 |
| --- | --- | --- | --- |
| 测试失败自动修复循环 | 代码生成后自动运行测试，失败时分类、修改、重跑，最多两轮 | 基线：人工修复时间、一次通过率；验收：自动修复成功率、重试次数、引入新失败数量 | 超过次数停止；核心模块要求人工 review |
| 客服答案纠错循环 | 将质检失败答案转成 prompt、知识库或规则修复任务，再回归验证 | 基线：质检失败率、重复错误数；验收：同类错误下降、修复样例通过率 | 不自动发布新话术；需质检负责人审批 |
| RAG 检索优化循环 | 对无引用、错引用和低满意答案分类，调整切片、排序或过滤规则 | 基线：引用错误率、无答案率；验收：引用准确率提升、召回率变化、延迟变化 | 提升召回不能牺牲安全过滤 |
| Agent 失败恢复循环 | 对工具失败、权限失败、参数失败建立恢复策略和升级规则 | 基线：任务失败率、人工接管率；验收：可恢复失败比例、误重试次数、升级准确率 | 权限失败不得自动绕过；重复失败及时停止 |
| Prompt 优化循环 | 将评测失败样例生成候选 prompt，跑 Harness 后选择版本 | 基线：当前 prompt 分数和失败分布；验收：目标指标提升、风险样例不退化 | 保留回滚版本；候选 prompt 必须人工审阅 |
| 知识库更新循环 | 根据搜索无结果、重复咨询和过期文档自动生成补充任务 | 基线：无结果搜索率、过期条目数；验收：无结果率下降、重复咨询下降、更新采纳率 | 只生成更新建议，不自动发布未经确认内容 |

## 数据记录模板

| 字段 | 说明 |
| --- | --- |
| 业务流程 | 说明该场景嵌入哪个真实流程，例如客服、法务、研发、销售、运维 |
| 基线数据 | 试点前人工流程的耗时、质量、成本、风险事件或满意度 |
| 样例来源 | 工单、合同、日志、代码、会议纪要、CRM、报销单等 |
| 样例规模 | 试点样例数量、抽样方式、脱敏方式和覆盖业务类型 |
| LLM 介入点 | 生成、抽取、分类、检索、工具调用、评测或修复 |
| 质量指标 | 正确率、召回率、引用准确率、质检通过率、人工采纳率 |
| 效率指标 | 平均处理时长、等待时间、返工次数、自动完成比例 |
| 成本指标 | 模型调用成本、人工复核成本、工具运行成本 |
| 风险指标 | 幻觉率、越权访问次数、敏感信息泄露、错误升级或错误拒答 |
| 决策结论 | 扩大试点、继续观察、回滚、补数据或改流程 |
