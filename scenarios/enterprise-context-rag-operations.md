# 企业 Context 与 RAG 场景

Context 工程的企业价值不在“塞更多资料”，而在于把来源、版本、权限、引用和评测做成稳定流程。

## 适用边界

- 适合：制度问答、售后知识库、研发代码库问答、销售资料助手、投研资料整合。
- 不适合：文档版本混乱、权限不可控、无法记录引用来源的问答系统。
- 关键判断：答案是否能追溯到具体资料版本和引用片段。

## 资料地址

| 资料 | 地址 | 用途 |
| --- | --- | --- |
| RAG 论文 | https://arxiv.org/abs/2005.11401 | 理解检索增强生成的基础结构 |

## 场景 1：制度知识库问答

| 项目 | 操作方案 |
| --- | --- |
| 资料范围 | HR、IT、财务、法务制度；每份资料必须带部门、地区、版本、生效日期、废止日期 |
| Context 装配 | 先按权限过滤，再检索，再按版本和生效日期重排，最后生成带引用答案 |
| 数据口径 | 引用命中率、过期制度拦截率、无依据回答率、人工升级率、员工满意度 |
| 验收方式 | 建立“必须引用文档”和“禁止引用文档”的问题集，单独统计冲突制度问题 |
| 资料支撑 | Google Cloud RAG Engine overview + OpenAI File search |
| 风险控制 | 冲突制度必须提示人工确认；过期制度不能进入上下文 |

## 场景 2：售后工单上下文装配

| 项目 | 操作方案 |
| --- | --- |
| 资料范围 | 客户档案、历史工单、产品手册、SLA、当前问题、已知故障公告 |
| Context 装配 | 对客户隐私字段脱敏；按当前问题匹配手册和历史相似工单；输出引用链 |
| 数据口径 | 首次响应时长、重复追问次数、一次解决率、误引用率、跨客户资料混入次数 |
| 验收方式 | 抽样 200 条历史工单做回放，比较人工处理路径和模型引用来源 |
| 资料支撑 | Nubank customer support agents paper，用于参考客服 Agent 的离线到线上评测框架 |
| 风险控制 | 跨客户资料不得进入上下文；高价值客户和投诉工单单独设阈值 |

## 场景 3：研发代码库问答

| 项目 | 操作方案 |
| --- | --- |
| 资料范围 | 代码、README、测试、Issue、PR、提交记录、架构决策记录 |
| Context 装配 | 按仓库、模块、最近变更、测试覆盖和调用关系组织上下文 |
| 数据口径 | 文件引用准确率、新人定位文件耗时、建议可执行率、错误 API 引用率 |
| 验收方式 | 用真实历史问题建立任务集，例如“定位订单状态流转入口”或“解释某个配置来源” |
| 资料支撑 | OpenAI File search + ReAct 论文：https://arxiv.org/abs/2210.03629 |
| 风险控制 | 不能直接合并生成代码；需要开发复核和测试通过记录 |

## 场景 4：销售知识助手

| 项目 | 操作方案 |
| --- | --- |
| 资料范围 | 产品资料、价格政策、竞品对比、客户行业资料、合规禁用语 |
| Context 装配 | 先匹配客户行业和销售阶段，再选择可公开资料和内部政策资料 |
| 数据口径 | 资料查找时长、方案准备时长、引用覆盖率、错误事实率、销售采纳率 |
| 验收方式 | 选 20 个历史销售机会重放，检查方案是否引用正确版本资料 |
| 资料支撑 | OpenAI Prompt engineering + OpenAI File search |
| 风险控制 | 价格、折扣、交付承诺必须引用官方政策，不能使用过期资料 |

## 场景 5：投研资料整合

| 项目 | 操作方案 |
| --- | --- |
| 资料范围 | 公告、财报、研报、新闻、行业数据、内部笔记 |
| Context 装配 | 对来源分层：一手公告和财报优先，媒体报道只作为线索 |
| 数据口径 | 引用覆盖率、事实核验通过率、来源等级分布、观点/事实混淆次数 |
| 验收方式 | 让模型输出“事实、来源、观点、待核验问题”四段，不允许把二手报道写成事实 |
| 资料支撑 | RAG 论文 + 企业内部投研资料库 |
| 风险控制 | 投资建议必须人工确认；模型只做资料整理和问题发现 |

## 最小数据表

| 字段 | 说明 |
| --- | --- |
| answer_id | 答案 ID |
| query | 用户问题 |
| corpus_version | 知识库版本 |
| retrieved_docs | 检索到的文档 ID 和版本 |
| cited_spans | 最终引用片段 |
| missing_context | 人工认为缺失的上下文 |
| answer_grade | 正确、部分正确、无依据、错引用、过期引用 |
| action | 补文档、调切片、调排序、改权限、转人工 |
