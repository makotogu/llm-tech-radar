# Eval Harness

## 目标

验证一套评测框架是否能比较不同 Prompt、Context 或 Agent 策略，并给出可复现结论。

## 适用专题

- Harness 工程
- Prompt 工程
- Context 工程
- Agent 工程
- Loop 工程

## 输入

- 测试集：固定样例、期望行为、标签和元数据。
- 候选策略：例如 prompt v1 与 prompt v2。
- Grader：规则评测、人工评审或 LLM-as-judge。

## 输出

- 分数表。
- 失败分类。
- 样例级 trace。
- 结论：推荐哪个策略，为什么。

## 流程

1. 固定测试集和版本号。
2. 对每个候选策略运行同一批样例。
3. 保存输入、上下文、工具调用、输出和成本。
4. 使用 grader 评分。
5. 抽样人工复核。
6. 生成对比报告。

## 验收标准

- 同一版本可重复运行。
- 评测结果能定位失败来源。
- 报告包含平均表现和关键失败样例。
- LLM-as-judge 结果至少有抽样人工校验。

## 失败模式

- 只看平均分，忽略高风险失败。
- 每次实验更换测试集。
- Grader 标准不稳定。
- 没有保存上下文和 trace。

## 后续实验

- 增加成本和延迟指标。
- 为 Agent 增加过程级评分。
