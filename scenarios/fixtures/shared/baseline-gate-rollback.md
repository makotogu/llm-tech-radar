# 共用附录：基线、门禁与回滚

企业试点立项时，把本附录字段填进试点记录。细分场景可增删列，但不要省略基线、门禁和回滚。

## 基线指标表头

| 字段 | 说明 |
| --- | --- |
| process_name | 业务流程名称 |
| risk_level | L1–L5 |
| baseline_period | 基线统计周期 |
| volume | 周期内样本量或工单量 |
| quality_metric | 质量指标名与口径 |
| quality_baseline | 试点前数值 |
| efficiency_metric | 效率指标名与口径 |
| efficiency_baseline | 试点前数值 |
| cost_metric | 成本指标名与口径 |
| cost_baseline | 试点前数值 |
| risk_metric | 风险指标名与口径 |
| risk_baseline | 试点前数值或事件数 |

## 上线门禁清单

| 检查项 | 通过标准 | 负责人 | 结果 |
| --- | --- | --- | --- |
| 样例集就绪 | ≥50 条脱敏样例，覆盖主路径与边界 | 业务/数据 |  |
| 基线可解释 | 质量/效率/成本/风险四类至少各有一项 | 数据口径 |  |
| 权限与数据分级 | 明确可读字段、禁止字段、租户隔离 | 安全 |  |
| 评测报告 | 固定集跑通，高风险类别单独门槛 | 技术 |  |
| 人工接管 | 升级条件、值班人和 SLA 已写明 | 业务 |  |
| 回滚方案 | 可在约定时间内切回旧流程或旧版本 | 技术 |  |
| 供应商与模型条款 | 数据保留、训练使用、区域限制已核验 | 法务/采购 |  |

## 回滚条件

满足任一条件即回滚或暂停扩大试点：

- 高风险样例通过率低于门禁阈值。
- 出现越权工具调用、敏感信息泄露或无法追责的自动决策。
- 成本或延迟超出预算且无降级策略。
- 人工接管失效，或值班路径不可用。
- 灰度期间核心业务指标相对基线显著恶化且无法归因修复。
