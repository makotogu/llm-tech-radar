# Context 装配 RAG

## 目标

验证一个多来源上下文装配流程是否能稳定提供相关、可引用、不过载的上下文。

## 适用专题

- Context 工程
- Harness 工程
- Agent 工程

## 输入

- 用户问题：需要外部知识或历史状态的问题。
- 数据源：文档片段、网页、代码文件、工具结果、历史对话。
- 预算：最大 token、最大资料条数、最大检索轮数。

## 输出

- 最终回答。
- 引用清单。
- 本次上下文装配清单：来源、得分、是否进入上下文、原因。

## 流程

1. 解析用户问题，拆出实体、时间、约束和期望输出。
2. 从候选来源召回资料。
3. 对资料按相关性、可信度、新鲜度和冗余度排序。
4. 对长材料做压缩，保留关键事实和引用。
5. 组装最终上下文，并标记外部内容与系统指令边界。
6. 生成回答，并输出引用。

## 验收标准

- 关键结论有可追溯来源。
- 不相关资料不会进入最终上下文。
- 超出预算时能解释丢弃或压缩了哪些内容。
- 外部内容不会覆盖系统指令和安全规则。

## 失败模式

- 检索命中但没有覆盖关键事实。
- 压缩丢失限制条件或反例。
- 引用看似存在但不支持结论。

## 后续实验

- 对比关键词检索、向量检索和混合检索。
- 给外部资料注入恶意指令，验证隔离策略。
