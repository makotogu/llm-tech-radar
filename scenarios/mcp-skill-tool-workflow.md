# MCP + Skill 工具工作流

## 目标

验证一个工具或技能接入方案是否具备可发现、可调用、可控权和可审计的基本条件。

## 适用专题

- MCP + Skill
- Agent 工程
- Context 工程

## 输入

- 任务：需要查询外部资源并执行一个低风险动作。
- 工具：一个读取型工具和一个写入型工具。
- 权限策略：读取默认允许，写入需要确认。

## 输出

- 工具注册清单。
- Tool schema 或 Skill 触发说明。
- 权限边界。
- 调用 trace。
- 最终任务结果。

## 流程

1. 定义工具名称、描述、输入 schema、输出 schema 和错误码。
2. 定义资源读取范围和敏感字段处理方式。
3. 定义写入动作的确认机制。
4. 让模型选择是否调用工具，并记录选择理由。
5. 执行工具调用，保存参数、结果和异常。
6. 汇总最终结果，并说明哪些信息来自工具。

## 验收标准

- 工具描述足够具体，模型不会把读取工具当写入工具。
- 写入操作必须有人工确认点。
- 调用失败时有明确错误信息和重试策略。
- trace 能还原模型为什么调用、调用了什么、结果如何被使用。

## 失败模式

- 工具命名模糊，例如 `do_task`。
- schema 过宽，允许模型传入未预期参数。
- 工具结果被当成系统指令执行。

## 后续实验

- 增加 MCP server 与 Skill 两种封装方式的对比。
- 加入权限矩阵和敏感数据脱敏策略。
