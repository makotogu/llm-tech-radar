# Agent 任务执行器

## 目标

验证 Agent 是否能围绕一个多步骤任务进行计划、执行、观察、失败恢复和交付。

## 适用专题

- Agent 工程
- MCP + Skill
- Harness 工程
- Loop 工程

## 输入

- 任务：例如“阅读一个小型代码仓库，找出一个 bug 并给出修复方案”。
- 工具：文件读取、搜索、测试运行、补丁生成。
- 约束：不得执行破坏性命令，必要时请求确认。

## 输出

- 任务计划。
- 每步执行记录。
- 工具调用 trace。
- 失败恢复记录。
- 最终交付物。

## 流程

1. 把任务拆成可执行步骤，并标记风险。
2. 读取必要上下文。
3. 执行低风险工具调用。
4. 根据结果更新计划。
5. 遇到失败时分类：信息不足、工具失败、权限不足、判断错误。
6. 完成后输出结果和证据。

## 验收标准

- Agent 不跳过上下文读取直接行动。
- 每个工具调用都有目的和结果记录。
- 失败时能给出恢复策略，而不是重复同一动作。
- 高风险动作有确认点。

## 失败模式

- 计划和实际执行脱节。
- 工具调用后不读取结果。
- 失败后无限重试。
- 最终答案缺少证据。
