# 企业 Improvement Loop 场景

Loop 工程关注的是“失败如何变成下一次系统改进”，而不是让模型无限自我重试。

## 适用边界

- 适合：测试失败自动修复、客服答案纠错、RAG 检索优化、Prompt 优化、知识库更新。
- 不适合：没有失败分类、没有版本管理、没有回滚机制的自动改动。
- 关键判断：每个循环是否有停止条件和人工检查点。

## 资料地址

| 资料 | 地址 | 用途 |
| --- | --- | --- |
| Reflexion paper | https://arxiv.org/abs/2303.11366 | 参考语言反馈和后续尝试改进的 Agent 循环 |
| Self-Refine paper | https://arxiv.org/abs/2303.17651 | 参考生成、反馈、精修的迭代模式 |
| DSPy paper | https://arxiv.org/abs/2310.03714 | 参考用指标优化语言模型 pipeline 的方式 |

| OpenAI Evals | https://developers.openai.com/api/docs/guides/evals | 作为每次循环是否改进的评测门禁 |

## 场景 1：测试失败自动修复循环

| 项目 | 操作方案 |
| --- | --- |
| 触发条件 | 代码生成后测试失败，或 CI 中低风险测试失败 |
| 循环流程 | 分类失败，读取相关代码和测试，修改，重跑，最多两轮 |
| 数据口径 | 自动修复成功率、重试次数、引入新失败数量、人工接管率 |
| 验收方式 | 只在分支运行；核心模块和安全模块必须人工 review |
| 资料支撑 | Reflexion paper + OpenAI Evals |
| 停止条件 | 两轮失败、出现新失败、涉及高风险模块、测试无法复现 |

## 场景 2：客服答案纠错循环

| 项目 | 操作方案 |
| --- | --- |
| 触发条件 | 质检失败、用户差评、人工纠错、投诉升级 |
| 循环流程 | 标注失败类型，判断修 Prompt、补知识库还是改升级规则，再跑回归集 |
| 数据口径 | 同类错误下降、修复样例通过率、重复投诉下降、人工质检分 |
| 验收方式 | 只有回归集通过且质检负责人确认后才能发布 |
| 资料支撑 | Nubank customer support agents paper + OpenAI Evals |
| 停止条件 | 缺少证据资料、质检结论不一致、风险类别需要政策确认 |

## 场景 3：RAG 检索优化循环

| 项目 | 操作方案 |
| --- | --- |
| 触发条件 | 无引用、错引用、低满意、召回不足、引用过期 |
| 循环流程 | 先定位失败是切片、索引、召回、排序、权限还是答案生成问题 |
| 数据口径 | 引用准确率、召回率、无答案率、错引用率、延迟、成本 |
| 验收方式 | 每次只改一个变量，比较新旧策略在同一评测集上的差异 |
| 资料支撑 | Ragas + Google RAG Engine overview |
| 停止条件 | 提升召回但增加高风险错引用，或延迟成本超过预算 |

## 场景 4：Prompt 优化循环

| 项目 | 操作方案 |
| --- | --- |
| 触发条件 | 回归集失败、输出格式漂移、人工修改比例上升 |
| 循环流程 | 从失败样例生成候选 Prompt，跑评测，比较版本，人工审阅后灰度 |
| 数据口径 | 目标指标提升、风险样例是否退化、人工修改比例、输出格式稳定性 |
| 验收方式 | 必须保留旧版本、候选版本、评测报告和回滚路径 |
| 资料支撑 | Self-Refine paper + OpenAI Evals cookbook |
| 停止条件 | 风险样例退化、收益不足、人工审阅不通过 |

## 场景 5：知识库更新循环

| 项目 | 操作方案 |
| --- | --- |
| 触发条件 | 搜索无结果、重复咨询、过期文档、工单新增问题 |
| 循环流程 | 生成知识缺口，找到来源，提出更新 PR，回归问答集验证 |
| 数据口径 | 无结果率、重复咨询率、更新建议采纳率、过期条目数 |
| 验收方式 | 知识库 owner 审核；新文档必须进入版本索引和引用评测集 |
| 资料支撑 | OpenAI Evals + 企业内部知识库记录 |
| 停止条件 | 缺少权威来源、资料冲突、无法确定生效范围 |

## Loop 记录最小字段

| 字段 | 说明 |
| --- | --- |
| loop_id | 循环 ID |
| trigger | 触发原因 |
| failure_type | 失败类型 |
| baseline_metric | 修改前指标 |
| change | 本轮修改内容 |
| eval_result | 回归评测结果 |
| approval | 审批人和结论 |
| rollout | 不上线、灰度、全量、回滚 |
